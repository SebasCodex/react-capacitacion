1. ¿Qué problema concreto resolvió React con los hooks que los componentes de clase no podían resolver bien? No digas "es más simple" — da un ejemplo técnico concreto
Un problema real era reutilizar lógica entre componentes. Con clases, si querías compartir lógica (por ejemplo hacer un fetch o manejar formularios), tenías que usar HOCs o render props, que hacían el código difícil de leer y mantener.
Con hooks puedes crear funciones reutilizables como useFetch() o useForm() y usarlas en varios componentes sin cambiar su estructura. Eso hace que la lógica esté separada y sea más fácil de reutilizar.

Yo prefiero usar varios useState cuando los datos son simples, porque es más claro y cada estado es independiente.

3. En el Sprint 0 de JavaScript usaban clases de ES6. ¿En qué se parece un componente de clase de React a una clase de ES6 normal? ¿Qué agrega React.Component?
Se parece a una clase de ES6 porque usa class, tiene un constructor, métodos y puede manejar propiedades (this).
