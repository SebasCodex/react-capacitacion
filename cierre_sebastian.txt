1. ¿Qué hace main.jsx y qué pasaría si lo borras?
main.jsx es como el punto de inicio de toda la app. Ahí es donde React se conecta con el HTML (el div con id "root") y donde se renderiza el componente principal, normalmente App.

2. ¿Por qué PlatoCard va en components/ y Home en pages/?
PlatoCard es un componente pequeño y reutilizable. Sirve para mostrar un solo plato y se puede usar varias veces dentro de la app. Por eso va en components/.

3. ¿Por qué ahora sí funciona el import con Vite y antes no con CDN?
Antes, cuando usaba React con CDN, todo estaba en un solo archivo HTML y el navegador no podía manejar bien imports entre archivos como jsx o js

4. ¿Qué va en hooks/, services/ y context/?
Las carpetas hooks,services y context se van a usar más adelante cuando la app crezca. En hooks/ voy a guardar funciones reutilizables de React.
